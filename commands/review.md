# コードレビュー

`$ARGUMENTS` で指定されたファイル、ディレクトリ、またはgit diff範囲をレビューする。
引数がない場合は `git diff --staged` をレビュー対象とする。

## レビュー観点

以下の各カテゴリについてチェックし、発見した問題を重大度付きで報告する。

### 1. セキュリティ (CRITICAL)
- ハードコードされたシークレット、APIキー
- SQLインジェクション、コマンドインジェクション、XSS
- 安全でないデシリアライゼーション
- 不適切な権限チェック

### 2. 正確性 (HIGH)
- エッジケースの未処理（null, 空配列, 境界値）
- エラーハンドリングの欠如
- 競合状態（race condition）
- リソースリーク（未クローズのファイル/接続）

### 3. パフォーマンス (MEDIUM)
- N+1クエリ
- 不必要なメモリ確保・コピー
- ループ内の重い処理
- 不適切なデータ構造の選択

### 4. スタイル・保守性 (LOW)
- 不明瞭な命名
- デッドコード
- マジックナンバー
- 過度に長い関数（50行超）

### 5. テスト
- 変更に対応するテストの有無
- テストが意味のある検証をしているか

## 出力フォーマット

```markdown
## レビュー結果

### CRITICAL
- [ファイル:行] 説明

### HIGH
- [ファイル:行] 説明

### MEDIUM
- [ファイル:行] 説明

### LOW
- [ファイル:行] 説明

### 良い点
- 特筆すべき良い実装があれば記載

### サマリー
全体的な評価を1-2文で。
```
