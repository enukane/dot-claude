# コンテキスト保全サマリー

`/compact` を実行する前に、現在のセッション状態を構造化して要約する。
この要約を `/compact` 後の最初のメッセージとして貼り付けることで、コンテキストの喪失を最小化する。

## 出力フォーマット

以下の形式でサマリーを生成する:

```markdown
## セッション引き継ぎ

### 現在のタスク
（何をしているか、1-2文）

### 完了した作業
- （実施済みの変更をリスト）

### 変更したファイル
- （パスと変更内容の要約）

### 決定事項
- （設計判断とその理由）

### 残りのTODO
- （次にやるべきこと）

### 注意点・ブロッカー
- （ハマった点や未解決の問題）
```

## 手順
1. 会話履歴から上記の情報を抽出する
2. `git diff` で実際の変更を確認して正確性を担保する
3. `tasks/todo.md` が存在すればその状態も反映する
4. 構造化テキストをユーザーに提示する

ユーザーはこの出力をコピーし、`/compact` 実行後に貼り付けて使用する。
