# LaTeX 論文ワークフロー

`ai/rules/LATEX_WORKFLOW.md` の規約に従って論文の分析・支援を行う。

## サブコマンド

`$ARGUMENTS` に応じて動作を切り替える:

### `review`（デフォルト、引数なし）
1. カレントディレクトリまたは `$ARGUMENTS` で指定されたディレクトリから `main.tex` または `*.tex` を探す
2. `\input{}` / `\include{}` を辿って全体構造をマッピングする
3. セクションごとのワード数を概算する（`texcount` があれば使用、なければ簡易カウント）
4. 以下の構造レポートを出力:
   - 章立て一覧と各セクションの概要
   - ページ数の概算
   - 図表の数と参照状況
   - 改善が有効そうな箇所の提案

### `build`
1. ビルドコマンドを検出: Makefile → latexmkrc → デフォルト（latexmk -pdf）
2. ビルドを実行
3. エラー・警告を分析して報告

### `check-refs`
1. `.bib` ファイルを探す
2. 本文中の `\cite{}` と `.bib` エントリの整合性チェック:
   - 本文で引用されているが `.bib` にないキー
   - `.bib` にあるが本文で引用されていないキー
3. `\ref{}` / `\label{}` の整合性チェック:
   - 未定義の参照
   - 使用されていないラベル

### `outline <テーマ>`
1. 指定されたテーマで論文のアウトラインを提案
2. IEEE/IEICE/ACMなどの学会テンプレートに合わせた章立て
3. 各セクションの推奨ワード数を配分

## 注意
- 著者の文体を尊重する。勝手にリライトしない
- 変更提案は具体的に（どのセクションの何行目付近か）
- 日本語と英語の論文両方に対応
